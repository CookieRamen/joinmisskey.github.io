Misskeyでは投稿を構文を使うことで装飾することができる。
この構文はMisskey Flavored Markdown (MFM)と呼ばれる。
その名の通りMarkdown構文を基にしているが、利用できる構文はMarkdownとは大幅に違う。

投稿だけでなく、プロフィールの自己紹介やメッセージにも利用できる。

以下に利用できる構文を示す。

太字

**～**で囲うことで太字で表示できる。

Text

Preview

標準の太さです。
太字で表示してみました！

中央揃え

[code
]と#[code
]とで囲った文字は中央揃えになる。複数行に渡って適用可能。

動く文字

文字にアニメーションを付けることができる。

ただし、文字が動くのは1箇所につき10文字以内で3箇所までという制限がある。

モーション

三重カッコで囲うか<motion>と</motion>とで挟むことで、条件を満たすと動く(文字サイズ・太さは通常)。タグ型の<motion>は顔文字を動かしたいときに便利。

ビッグ

文字を`***`で挟むと、通常の1.5倍の大きさの太字で表示され、条件を満たすと動く。

Text

Preview

動きます！
動く？
こいつ……動くぞ！

コード

プログラミングのコードやアスキーアートを記述するとき便利な構文。
いくつかの文字に色が付く。フォントは等幅文字で表示される。

ブロック

以下の例のように```…```で囲うと、その間の行がコードとして表示される。

行内

行中で`…`で囲うとコードとして表示される。
変数や値、ファイル名などを解りやすく表示するのに使うといいのではないだろうか。
数字、特定の文字列、記号などは彩色される。

ただし「´」を入れるとインラインコード構文として認識されなくなる。

Text

Preview

/*
 *Code (block)
 */
// ソースコードやアスキーアートが捗る？
長い行があるときはスクロールして表示されます。ちぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇぇん
行内コードのmfm構文です。

"…"で囲うか、行頭を> とすることで、引用として表示される。

検索

○○ Search ○○ 検索 ○○ [Search] ○○ [検索]という行は例のような検索ボックスになる。

Preview

リンク

[link](url)という記述は、urlに飛ぶlinkという表示のハイパーリンクとして表示される。
?[link](url)とすると、投稿下部に表示されるリンクの詳細が表示されなくなる。(残念ながら例では変わりないが…)

URLをそのまま貼り付けるとそのままリンクとして認識される。

タイトル

【…】または[…]という行は例のようにタイトルとして表示される。

Preview

ねこです
きいてますか。
